# Autoresearch Loop

核心原则：有效就留，无效就扔，始终以单一主指标驱动决策。

## Workflow

1. 先跑通 baseline，并记录基线指标。
2. 每轮只做一类改动（优先模块改造，再考虑参数）。
3. 训练完成后提取主指标并与当前最优比较。
4. 提升则保留（keep），无提升或失败则回退（discard/repair）。
5. 当指标进入平台期且效果仍差，触发模型结构重构而不是继续硬调参数。

## Decision Rules

- Keep:
  - 主指标优于当前最优，或
  - 指标持平但复杂度更低/稳定性更好。
- Discard:
  - 指标下降；
  - 崩溃、超时、OOM；
  - 复杂度增加但收益不明显。

## Constraints

- 评估标准保持一致，保证可比性。
- 资源预算固定，避免“靠更多资源假提升”。
- 优先保证可复现和可回滚。
