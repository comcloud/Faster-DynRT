# 多模态图文讽刺/情感检测模型改造指导

该项目是图文讽刺检测，与多模态情感分析在改造逻辑上高度一致。请在每次改造时优先考虑以下方向，并保持外部接口与张量契约不变。

## 1. 特征对齐
- 优先增强文本和图像特征空间的一致性。
- 除最后层特征外，可引入中间层特征做多级融合。
- 可考虑在训练目标中加入跨模态对比约束，提升正负样本区分度。

## 2. 融合机制
- 避免仅使用简单拼接。
- 优先尝试跨模态交叉注意力或门控残差融合，提升图文交互质量。
- 需要控制计算量，避免明显增加显存开销。

## 3. 细粒度关联
- 强化文本 token 与图像局部区域之间的对应关系。
- 重点提升对冲突样本、反讽样本的表示能力。

## 4. 训练与鲁棒性
- 当指标进入平台期时，优先做结构改造而不是只调超参数。
- 保持训练稳定，避免引入容易发散的改动。
- 若改造失败，给出小步可回滚版本。

## 5. 输出要求
- 仅修改给定目标文件。
- 不改文件路径与外部调用接口。
- 变更需可训练、可复现，并优先提升 `valid_acc` 与 `valid_f1_macro`。
